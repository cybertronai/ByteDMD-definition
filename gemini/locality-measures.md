Based on the provided article *"A Relational Theory of Locality"* by Yuan et al., the authors introduce a unified mathematical framework that formalizes how different definitions of locality relate to and convert into one another.

Here is an overview of the locality measures provided, how to calculate them for matrix multiplication workloads, and the primary differences you will observe between **Naive Matrix Multiplication** (standard $i, j, k$ nested loops) and **Tiled/Blocked Matrix Multiplication** (processing the matrices in $T \\times T$ sub-blocks).

### ---

**1\. Overview of the Locality Measures**

The paper structures locality into four categories, ranging from exact execution orderings to abstract hardware models:

* **Access Locality (Per-reference Metrics):** Measures locality for every individual memory access.  
  * *Singleton Locality:* The raw, literal memory address execution trace.  
  * *Sequence Locality:* Retains the chronological ordering of reuses. Key sequences are the **Reuse Interval (RI)** (the logical time/number of total accesses since the exact same datum was last accessed) and the **Reuse Distance (RD)** (the number of *distinct* data addresses accessed since its last use, also known as LRU stack distance).  
  * *Histogram Locality:* Unordered, compact summaries that bin the RI or RD sequences into frequency distributions (e.g., $k$-sublog histograms).  
* **Timescale Locality (Time-window Metrics):** Mathematical functions measuring data demand as a function of logical time (window size $x$).  
  * *Working Set ($s(x)$):* Uses the iterative "Denning recursion" to find the average working set size as a time window grows.

  * *Footprint ($fp(x)$):* The average number of distinct elements accessed across all execution time-windows of length $x$.  
* **Cache Locality (Space-based Metrics):** Measures cache performance directly using cache capacity $c$ as a parameter.  
  * *Miss Ratio Curve (MRC, $mr(c)$):* The percentage of accesses that miss in a fully associative LRU cache of size $c$.  
  * *Fill / Eviction Time:* The average time for an empty cache of size $c$ to incur $c$ misses, or the time between a block's last access and its eviction.  
* **Frequency Locality (Hotness):** The simplest metric; the average access frequency per data element ($n/m$).

### ---

**2\. Calculations and Workload Differences (Naive vs. Tiled)**

Assume we are multiplying two $N \\times N$ matrices ($C \= A \\times B$). Here is how to calculate the quantities through the paper's conversion pipeline and the theoretical differences between Naive (NMM) and Tiled (TMM) operations.

#### **A. Frequency Locality (Hotness)**

* **How to Calculate:** Count the total number of memory accesses generated by the loops and divide by the total number of distinct memory addresses.  
* **The Difference: None.** Both NMM and TMM perform the exact same mathematical operations: $O(N^3)$ total accesses over $3N^2$ distinct data elements. As Section 2.6 of the paper points out (the "Snir-Yu limit"), frequency completely ignores the *recency* and *order* of accesses. Evaluating by hotness alone will falsely conclude both workloads have identical locality, masking the benefits of loop tiling.

#### **B. Access Locality (RI / RD Histograms)**

* **How to Calculate:** First, generate an array of memory addresses accessed by the loops.  
  * *For RI:* Traverse the trace using a hash table to record the last-accessed timestamp of each address. Subtract this from the current time.  
  * *For RD:* Simulate an LRU stack. At each access, find the element in the stack, record its depth (this is the RD), and pull it to the top. Count occurrences of these distances to build your RD Histogram.  
* **The Difference:**  
  * *Naive:* NMM inner loops sweep entirely across rows of $A$ and columns of $B$. An element in $B$ is not reused until the outermost loop iterates, meaning the entire matrix $B$ is accessed between reuses. The **RD Histogram** will exhibit massive concentration spikes at huge distances, specifically $O(N)$ and $O(N^2)$.

  * *Tiled:* TMM restricts operations to $T \\times T$ blocks. Data from $A, B$, and $C$ are reused rapidly *within* the tile before moving to the next block. The **RD Histogram** shifts drastically to the left, showing bounded distances peaking tightly around $RD \\le 3T^2$.

#### **C. Timescale Locality (Footprint $fp(x)$)**

* **How to Calculate:** Instead of inefficiently simulating sliding windows over a massive trace, use the **Xiang formula** (Equation 3\) or the paper's linear-time Additive/Incremental formulas (Eqs. 8 & 9). You simply input your compact RI histogram, alongside the first and last access times of your data, to calculate the footprint $fp(x)$ directly.  
* **The Difference:**  
  * *Naive:* The footprint curve $fp(x)$ will grow rapidly and almost linearly as the timescale window $x$ grows. Because NMM continuously sweeps across new columns without short-term reuse, the working set expands uninhibited until it hits the size of whole rows or full matrices.  
  * *Tiled:* The footprint curve $fp(x)$ will exhibit a distinct **plateau**. It will grow to $3T^2$ (the size of the three active tiles) and stay completely flat for a massive duration of logical time ($O(T^3)$ accesses) because the working set is strictly bounded to the active tile.

#### **D. Cache Locality (Miss Ratio Curve)**

* **How to Calculate:** Map Timescale Locality directly to Cache Locality using the **Denning-HOTL conversion** (Equation 6: $mr(c) \= \\Delta fp(x) |\_{fp(x)=c}$). The miss ratio for a cache size $c$ is simply the finite difference (growth rate) of the footprint function when the footprint equals the cache size.  
* **The Difference:**  
  * *Naive:* The MRC will sit stubbornly high (near 100% capacity misses for un-retained dimensions) for small and medium caches. It will only drop significantly when the cache size $c$ becomes massively large—enough to hold at least a full matrix row ($O(N)$), hitting near-zero only when $c \\ge O(N^2)$.  
  * *Tiled:* The MRC will feature a precipitous, cliff-like drop much earlier on the x-axis. As soon as the cache size $c$ reaches $\\ge 3T^2$, the miss ratio plummets to near-zero. This explicitly demonstrates how TMM eliminates capacity misses by tuning the $T$ parameter to fit naturally within the hardware's L1/L2 cache capacity.
